# 拼多多自动上架工具

当前已支持 ERP 账号密码登录：价格册查询会优先复用 `erp.storage_state`，如果登录态失效或被踢回登录页，会自动使用 `erp.username` / `erp.password` 登录，登录成功后保存新的 `storage_state`，再继续读取价格册价格。

## 桌面双击启动

运行 `create_desktop_shortcut.ps1` 会在当前用户桌面创建 `拼多多自动上架` 快捷方式。以后双击这个快捷方式，会先检查本地工作台是否已经启动；如果已启动且代码未变化，就直接打开 `http://127.0.0.1:8765/`。如果工作台启动后又更新过 `desktop_tool.py`、`main.py`、`skills` 下的 Python 逻辑或 `templates` 下的 YAML 模板，启动器会只识别并重启占用 8765 端口的本项目 `desktop_tool.py --no-open` 旧进程，再加载磁盘上的最新代码；如果工作台未启动，则会在后台以 `--no-open` 方式运行，并在工作台可访问后只打开一次网页。

## ERP 优质价规则

自动上架时使用“订单 > 价格册 > 优质价”里的价格作为基础价，再乘以商品 `meta.yaml` 里的 `price_multiplier` 填到平台售价。

尺寸图文件名要用于精确匹配优质价：

- 文件名里包含完整型号，例如 `8065-25古铜色.jpg`；尺寸和颜色可以直接连写，也支持 `8065-25-古铜色.jpg` 这种带分隔符的写法。
- 程序会先搜索基础型号 `8065`。
- 如果基础型号没有返回结果，会再用完整型号重试，例如 `8065-25`。
- 搜索结果里会精确匹配 `8065-25`，不会误拿 `8065-20` 或 `8065-30`。
- 如果同一型号有多个颜色，会继续按文件名里的颜色匹配，例如 `古铜色`。像 `8264-96铜本色.jpeg`、`8264-128古铜色.jpeg`、`8264-160铬.jpeg` 会分别识别为完整型号 `8264-96`、`8264-128`、`8264-160` 及对应颜色，不需要批量改名。

## 配置 ERP

复制 `config.example.yaml` 为 `config.yaml`，填好 ERP 地址和页面选择器。账号密码建议用环境变量：

```powershell
$env:ERP_USERNAME="你的ERP账号"
$env:ERP_PASSWORD="你的ERP密码"
```

`config.yaml` 中保持：

```yaml
erp:
  username: "${ERP_USERNAME}"
  password: "${ERP_PASSWORD}"
```

需要用 F12 填好的选择器：

- `username_input`: ERP 登录页账号输入框
- `password_input`: ERP 登录页密码输入框
- `login_submit`: ERP 登录按钮
- `search_input`: 价格册搜索框
- `search_submit`: 搜索按钮，可留空，留空会按 Enter
- `price_cell`: 价格显示元素
- `login_error`: 登录失败提示，可留空
- `wait_timeout_ms`: ERP 页面和价格接口等待时间，建议保持 `30000`，避免优质价接口偶发慢响应时过早失败

真实账号、密码、`states/*.json` 不要提交到 git。

## 图片空间模式

当前工作台优先按拼多多后台“图片空间路径”生成上架包，不再要求每个商品都准备本地商品文件夹。

正常使用时填写：

- `价格倍数`：例如 `1.8`，用于把 ERP 优质价换算成平台拼单价。
- `价格尾数`：选择 `8` 或 `9`；程序会先四舍五入到两位小数，再把小数点后第 2 位调整为所选尾数。
- `材质`：例如 `黄铜`、`锌合金`、`铝合金`。
- `图片空间路径`：单任务可填写 `2026/8105-8255`；批量任务每行填写一个路径。

### 批量任务队列

需要连续处理多个商品时，在“图片空间路径”中每行填写一个路径。每条路径下方会自动生成一张任务配置卡，可以分别填写价格倍数、选择价格尾数和材质，并为该任务单独填写商品标题；不同任务的定价、材质与标题互不覆盖。标题留空时，程序会在该任务生成上架包后自动使用推荐标题；手动标题会按拼多多 60 字节限制即时计数并在入队时再次校验。页面顶部的价格倍数、价格尾数和材质只作为新增批量任务的默认值，需要统一调整时可点击“把上面的默认值应用到全部任务”，应用后仍可逐条修改，且不会覆盖已填写的手动标题。确认每张卡片后点击“批量加入并自动执行”。程序会逐项执行以下流程：读取图片空间、查询 ERP 优质价、确定该任务标题、打开拼多多发布页、自动填充并保存草稿。前一个任务确认结束后才会开始下一个任务，不能同时覆盖当前上架包。

批量队列会保存到 `.tmp_tool/batch_listing_queue.json`，刷新工作台不会丢失待执行和失败记录，工作台重启后也会继续未完成的队列。每项只有在插件报告“草稿已保存”后才记为成功；读取素材、ERP 查价、类目/详情填充报错，或自动填充结束但草稿未保存，都会记为失败并自动跳过。单项等待插件超过 45 分钟也会按失败跳过，避免整个队列永久卡住。

工作台的“批量任务队列”会显示总数、成功数、失败数和每项失败原因。批次全部完成且存在失败项时，页面会弹窗提醒；如果已允许本地工作台发送浏览器通知，也会使用系统通知提醒。执行期间需要保持 Chrome 已登录拼多多后台，首次遇到登录或滑块验证仍需人工完成。

图片空间目标文件夹下仍需包含 3 个子文件夹：

- `主图`
- `详情页`
- `尺寸图`

程序会用图片空间“尺寸图”的文件名生成 SKU，并按文件名匹配 ERP 优质价。本地商品文件夹现在只作为兼容旧流程的可选输入：如果该目录存在 `meta.yaml`，会继续读取其中的类目、材质、库存等默认值；如果不存在，也不会阻止生成上架包。

`主图` 文件夹里可以同时放主图图片和视频文件。图片会上传到商品轮播主图；视频文件会优先作为主图区域里的商品视频处理：插件只会从对应的视频上传区域打开“图片空间上传”，不会再使用规格图片入口；弹窗打开后会把顶部素材类型从“图片”强制切换成“视频”，确认页面出现“仅展示上传成功的视频”后，才定位到当前 `图片空间路径/主图`，按完整文件名优先匹配视频卡片并点击右上角的选择框。只有底部“已选 0 张”变成大于 0 后才会点“确认”，并继续等待弹窗关闭、发布页出现视频预览；缺少任何一个确认信号都不会把视频记为成功，失败时会自动点击“取消/关闭”，不再把图片空间弹窗遗留在发布页。文件名带 `9-16`、`讲解` 或 `竖屏` 的视频优先用于商品讲解视频，其余视频优先用于商品视频；如果只有 1 个视频，两个位置复用同一个视频。

品牌不在本地模板里写死。不同店铺对应不同品牌时，发布页选完类目后由拼多多后台显示该店铺可选品牌；工作台不会默认填写“梵居匠”，避免跨店铺选错品牌。

尺寸图文件名里的 `单孔`、`吊坠` 等安装/规格描述会用于 ERP 价格册型号后缀。例如 `2715云栖-铬色-单孔` 会用 `2715-单孔` 查询价格册，并用 `铬色` 匹配价格册颜色 `铬`；如果文件名带括号尺寸，例如 `6601-古铜色-单孔（24）`，括号数字会按图片标注尺寸处理，不作为优质价型号，仍用 `6601-单孔` 查价。没有安装后缀、只有尺寸数字时，会按尺寸查价，例如 `8256-古铜色-96尺寸图` 查 `8256-96`；如果旧缓存已经生成了 `6601-24` 这类查询，价格册没有该型号但有唯一的 `6601-单孔` 或 `6601-吊坠` 时，也会回退到对应安装后缀。商品属性“孔距”是单选项，也只从本次尺寸图生成的 SKU 判断：只要识别到 `单孔` 就只选择“单孔”；没有单孔时，只选择识别到的最小数值孔距，例如同时存在 `96`、`128`、`192` 时只选择 `96mm`，绝不再拼接或同时填写多个孔距。商品属性“外形”与该孔距同步推导：单孔只选择“球形”，`96mm` 等数值孔距只选择“条形”；没有识别出孔距时，不使用模板默认的孔距或外形。部分图片命名颜色会按 ERP 价格册颜色归一：`钛银` 对应系统 `亮镍`，`亮金` 对应系统 `玫瑰金`，`PVD金` 对应系统 `铬PVD`，`铬色` 对应系统 `铬`。如果尺寸图文件名写的是 `亮镍`，拼多多规格名会显示为 `钛银`，但 ERP 查价和规格编码仍使用系统颜色 `亮镍`。

一个店铺、一个商品链接下的全部 SKU 固定按“颜色 → 型号 → 孔距”三级顺序创建和填写。公司颜色顺序为：`古铜色 → 铬色 → 钛银 → 铜拉丝 → 哑镍拉丝 → 黑色 → 亮金 → 铜本色`；必须先排完一个颜色下的全部型号和孔距，再进入下一个颜色。同一颜色内按型号编号从小到大，同一型号内按孔距数字从小到大，`单孔` 视为最小孔距排在数值孔距前；未列入固定顺序的颜色不会丢失，统一排在上述颜色之后。ERP 颜色别名也按页面展示颜色归组：`铬` 按 `铬色`、`亮镍` 按 `钛银`、`玫瑰金` 按 `亮金` 排序。例如两个型号同时含古铜色和铜本色时，顺序为 `古铜色 8155-96 → 古铜色 8155-128 → 古铜色 8168-96 → 古铜色 8168-128 → 铜本色 8155-96 → 铜本色 8155-128 → 铜本色 8168-96 → 铜本色 8168-128`，不会按型号优先把颜色拆开。点击“打开发布页开始填充”或“填规格价格测试”时，发布页商品级“商品编码”会自动填写工作台里的价格倍数，例如 `1.6`；满件折扣默认填写 `9.9` 折；商品参考价固定填写为“所有 SKU 中最贵的单买价 + 1 元”。

部分店铺（例如利德仕旗舰店）进入发布流程后，会先出现“商品主图 / 商品标题 / 商品分类 / 下一步，完善商品信息”的发布前信息页，也有店铺使用“以图发品 / 上传商品轮播图 / 选择商品分类 / 下一步，完善商品信息”且完全没有标题输入框。工作台会优先根据页面控件识别这些页面，即使网址仍是详情页地址也不会误判；任务轮询发现发布前信息页后，会等待页面结构稳定，并为初始化期间短暂出现的旧类目树保留 6 秒观察期，再领取上架任务。有标题框时严格按“激活上传图片入口并上传/复用主图 → 填写并校验标题 → 选择类目 → 下一步”的顺序执行；没有标题框的以图发品页只校验主图，页面已经显示目标“已选分类”时跳过重复选类目并直接点击下一步。页面真实存在的主图或标题任一失败都会立即停止，传统类目选择失败时也不会再点击“下一步”；20 秒内没有稳定加载时不会领取任务。

更新本地插件代码后，必须先在扩展管理页重新加载插件，再刷新已经打开的拼多多发布页；只重新加载插件而不刷新旧发布页时，旧页面会继续运行之前已经注入的 `content.js`。可先打开插件弹窗核对版本号，再重新发起上架任务。

图片空间“主图”文件夹里的 `mp4`、`mov`、`webm`、`m4v` 会随上架包保留为视频素材。真实视频会从图片空间弹窗里选择已有素材，不再通过“本地上传”重新上传并等待进度条；弹窗会根据文件名定位视频卡片并勾选后确认。如果“主图”文件夹没有视频文件，插件才会用第 1 张主图在浏览器里生成短视频兜底：“商品视频”生成约 6 秒 1:1 视频，“商品讲解视频”生成约 12 秒 9:16 视频；这类临时生成视频仍需要走页面上传入口。

详情页会自动勾选“承诺”里的“假一赔十”，勾选“服务”里的“正品发票”，并把开票方式设置为“每申请单必开”。商品视频、商品讲解视频或这些承诺/服务任一未成功处理时，程序会停止自动保存草稿并在页面提示需要检查，避免把缺项草稿记为已完成。批量队列会优先显示具体失败步骤，例如“未打开图片空间弹窗”“未在指定主图目录选中视频”“弹窗没有确认关闭”或“发布页没有出现视频预览”，不再只显示笼统的“草稿未自动保存”。

每次自动点击并确认“保存草稿”后，工作台会追加一条历史记录到 `.tmp_tool/saved_draft_history.json`，包含保存时间、店铺名、店铺 ID、商品标题、商品链接或 `goods_id`、由“店铺 ID:商品 ID”组成的 `record_key`、商品文件夹、图片空间路径、ERP 型号、类目、规格数量、主图数量和详情图数量。工作台首页会显示“累计保存草稿”总数和最近 5 条记录，并直接露出店铺 ID、商品 ID 和记录键，方便核对这个自动上架工具已经帮哪些店铺保存过哪些商品草稿。
